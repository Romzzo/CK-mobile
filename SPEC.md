## 1. 히어로 배너

### 이미지 관리 규칙
- 저장 경로: `public/hero/`
- 파일명: `hero-01.jpg`, `hero-02.jpg` ... (확장자 jpg/png/webp 가능)
- 권장 사이즈: 1200×720 이상, 가로형 (세로가 너무 길면 모바일에서 잘림)
- 코드 활성화: `src/data/hero.ts`의 `HERO_IMAGES` 배열에 `/hero/파일명.jpg` 추가
- 홈 진입 시 배열에서 랜덤 1개 히어로 배경으로 표시

### 현재 등록된 이미지
`hero-01.jpg` ~ `hero-07.jpg` (7장)

### 이미지 업로드 방법 (봇 연동)
Discord 채널에서 이미지 파일 첨부 + 경로 입력 시 자동 저장됨.
- 예: 이미지 첨부 + `public/hero/hero-08.jpg` 입력 → 해당 경로에 저장
- 이미지만 첨부 시 원본 파일명으로 프로젝트 루트에 저장

### 히어로 카피
`일잘러의 필수 사이트` (변경 완료 2026-05-14)

## 2. 업데이트 페이지 (`/update`)

### 페이지 개요
주차별 신규 콘텐츠를 **테마 단위**로 묶어 보여주는 페이지.
각 테마는 제목이 있고 여러 개의 개별 콘텐츠로 구성됨.

### 레이아웃 구조

**상단 필터 바**
- 주차 선택 + 카테고리 선택 나란히 2개
- 탭 시 바텀시트 방식
- 주차 형식: `YYYY년 MM월 N주차` (월 1주차 = 해당 월 1일이 포함된 주, 주 = 일~토)

**주차 헤딩**
- `YYYY년 MM월 N주차` + 날짜 범위 (`MM.DD ~ MM.DD`) 한 줄

**테마 카드 그리드**
- 2열 그리드
- 이미지 비율: `aspect-[4/3]` (가로형 — 실제 사이트 기준)
- 카드 하단: 테마 제목 (1줄 truncate) + `□ N` 형태 콘텐츠 수 (우측 정렬)
- 탭 → 테마 상세 페이지로 이동 (`/update/detail/[seq]`)

**카테고리 목록**
전체 / 국내일러스트 / 해외일러스트 / 아이콘 / 포토 / AI이미지 / PPT / 폰트 / 3D / PNG / 합성·웹

### 데이터 구조

```ts
type Theme = {
  id: string;
  title: string;       // 테마 제목 (예: "Golden Brass")
  cover: string;       // 커버 이미지 URL
  count: number;       // 포함 콘텐츠 수
  category: string;    // 카테고리
  seq: number;         // 상세 페이지 이동용 ID (예: 71494)
  pcUrl: string;       // 연결 URL
}

type WeeklyUpdate = {
  id: string;
  year: number;
  month: number;
  week: number;
  dateRange: string;   // "05.10~05.16"
  themes: Theme[];
}
```

### 현재 구현 상태
- 데이터: 2023-01-01 ~ 2026-05-10 절차적 생성 (약 178주차)
- 카드 비율: `aspect-[4/3]` ✅
- 콘텐츠 수 아이콘: `Square` (□ N) ✅
- 바텀시트 방식 ✅

### 미구현
- 테마 카드 탭 시 `/update/detail/[seq]` 라우팅 (현재 PC URL 외부 링크)
- `Theme.seq` 필드 미포함 (더미 데이터에 추가 필요)

## 3. 테마 상세 페이지 (`/update/detail/[seq]`)

### 페이지 개요
실제 URL 패턴: `https://m.clipartkorea.co.kr/update/detail?seq=71494`
하나의 테마에 속한 개별 콘텐츠들을 그리드로 보여주는 페이지.

### 레이아웃 구조

**헤더**
- 좌: 뒤로가기(`<`) → `/update` 복귀
- 중앙: 테마 제목 (예: "그날의 외침")

**서브 헤더 (헤더 하단)**
- 좌: `N 것` (포함 콘텐츠 수)
- 우: 카테고리 + 날짜 (예: `국내일러스트 2026.05.14`)

**콘텐츠 그리드**
- 2열 그리드
- 카드: 이미지만 (제목 없음, 번호 없음)
- 이미지 비율: `aspect-square` (정사각형)
- 이미지 우하단: 담기 아이콘 버튼 (작은 오버레이)
- 탭 → 개별 콘텐츠 상세로 이동

### 라우팅
- Next.js: `/app/update/detail/[seq]/page.tsx`
- 또는 쿼리스트링: `/app/update/detail/page.tsx?seq=71494`

### 현재 구현 상태
- **미구현** (페이지 없음)

## 4. 좋아요한 콘텐츠 페이지 (`/like`)

### 페이지 개요
사용자가 좋아요한 **개별 콘텐츠** 목록 (테마 아님).
개별 콘텐츠는 제목 없음 — 번호(콘텐츠 코드)만 있음.

### 레이아웃 구조

**헤더**
- 타이틀: "좋아요한 콘텐츠"
- 서브타이틀: `N개` (전체 기준, 필터 무관)
- 우측: "편집" 버튼 (편집 모드 진입)

**유형 필터 바**
- 가로 스크롤 pill 형태
- 항목: 전체 / 일러스트 / 사진 / 아이콘 / AI이미지 / PPT

**콘텐츠 카드**
- 2열 masonry 그리드
- 이미지 비율: `aspectRatio` 데이터 기준 (`tall` → 3:4, `square` → 1:1, `wide` → 3:2)
- 카드 하단 항상 노출 (30px 이하): 좌-콘텐츠 번호 / 우-카테고리 뱃지
- 프리미엄 콘텐츠: 이미지 좌하단 `🔒 PRO` 오버레이 뱃지
- 탭: 상세 페이지 이동 (단일 탭, 예외 없음)

**편집 모드**
- 진입: 헤더 우측 "편집" 탭
- 편집 모드 시: 카드 좌상단 `-` 아이콘 노출, 탭하면 좋아요 해제
- 편집 모드 시 카드 탭 → 상세 이동 X
- 해제 확인 다이얼로그 없음 (즉시 제거)

**빈 상태**
- 하트 아이콘 + "좋아요한 콘텐츠가 없어요" + "콘텐츠 둘러보기" 버튼

### 데이터 모델 (개별 콘텐츠)
- 제목 없음
- 콘텐츠 번호(코드)만 표시
- 카테고리 있음
- 프리미엄 여부 있음

## 5. 바텀 네비게이션

### 현재 구성
홈 / 카테고리 / 검색 / 좋아요 / MY

### 논의 내용 (2026-05-14)

**카테고리 → 업데이트 교체 검토 중**
- 이유: 카테고리는 탐색 수단이지 목적지가 아님. 홈 CategoryCards에서 이미 접근 가능.
- 업데이트는 주기적 재방문을 유도하는 핵심 행동 — 실제 사이트도 상단 2번째에 위치.
- 조건: 홈 화면에서 카테고리 탐색 동선 보완 필요.

**이벤트 탭 논의**
- 인스타그램 광고 유입(모바일) 대상으로 마케팅 수단으로서 모바일웹 운영 목적.
- 이벤트는 상시 운영 예정 → 바텀 네비 배치 가능성 있음.
- 현재 결론: 업데이트 탭 우선, 이벤트 탭은 추후 결정.

**결정 보류 사항**
- 카테고리/업데이트/이벤트 중 최종 구성 미확정
- 좋아요를 MY 안으로 흡수하고 다른 탭 추가하는 구조도 검토 가능
